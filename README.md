## Описание работы

В данной лабораторной работе реализовано взаимодействие между Python, Go и Rust для решения задач сетевого взаимодействия и повышения производительности вычислений. Основное внимание уделено созданию TCP-сервера на Go, использованию Rust для создания высокопроизводительного Python-модуля, а также сравнению производительности трёх подходов.

## Выполненные задания

### Средней сложности

**Задание 1: TCP-сервер на Go + Python-клиент (Задание 5 по Методичке)**
*   Реализован простой TCP-сервер на языке Go (файл `zad1/server.go`), который обрабатывает входящие строковые запросы и возвращает ответ.
*   Создан Python-клиент (`zad1/client.py`), который подключается к Go-серверу и отправляет данные.

**Задание 2: Добавление горутины для фоновой обработки запросо (Задание 2 по Методичке)в**
*   В сервер добавлена **горутина** (`zad2/server_goroutine.go`), которая выполняет фоновую обработку запросов (например, имитацию длительной задачи), не блокируя основной поток обработки подключений.

**Задание 3: Rust-модуль для Python (Задание 10 по Методичке)**
*   Создана структура на Rust (`zad3/lib.rs`), которая экспортируется в Python как полноценный класс с помощью библиотеки `PyO3`.
*   Класс включает конструктор и методы для выполнения высокопроизводительных вычислений (например, математические операции или обработка данных), что позволяет Python-коду делегировать "тяжёлую" работу на Rust.

### Повышенной сложности

**Задание 4: Профилирование и сравнение производительности (Задание 5 по Методичке)**
*   Проведено сравнение производительности трёх решений для вычислительной задачи (например, обработка списка чисел):
    1.  **Чистый Python**.
    2.  **Python + Rust** (вызов скомпилированного модуля из задания 3).
    3.  **Python + Go** (запрос к TCP-серверу из заданий 1-2).
*   Для профилирования использовался модуль `timeit` и утилиты мониторинга. Результаты сравнения и выводы представлены в файле `zad4/comparison_results.md` или в виде графика.

**Задание 5: CI/CD сборка и публикация на PyPI (Задание 3 по Методичке)**
*   Настроен автоматизированный процесс сборки Rust-модуля с использованием `maturin`.
*   Конфигурация CI/CD (например, через GitHub Actions) описана в файле `.github/workflows/CI.yml`.
*   Реализована автоматическая публикация собранного пакета в репозиторий PyPI (или TestPyPI) при создании тега в репозитории. Файлы конфигурации находятся в папке `zad5`.

### Структура проекта

- **`.github/workflows/CI.yml`** — CI/CD автоматическая сборка и публикация на PyPI

- **`zad1/`** — TCP-сервер на Go и Python-клиент
  - `server.go` — реализация TCP-сервера
  - `client.py` — клиент на Python

- **`zad2/`** — Добавление горутины для фоновой обработки
  - `server_goroutine.go` — сервер с горутиной

  - `Cargo.toml` — манифест Rust
  - `lib.rs` — структура на Rust для экспорта в Python
  - `python/test_rust_module.py` — пример использования

- **`zad4/`** — Сравнение производительности
  - `benchmark.py` — скрипт профилирования

- **`zad5/`** — CI/CD и публикация на PyPI
  - `publish_config.md` — конфигурация публикации
  - `pyproject.toml` — сборка Rust-модуля с maturin

- **Корневые файлы**
  - `.gitignore` — игнорируемые файлы
  - `go.mod` — файл модуля Go
  - `PROMPT_LOG.md` — лог взаимодействия
